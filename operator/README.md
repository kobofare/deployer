# 升级脚本规划

此目录用于存放升级相关脚本。

## 目录说明

### `upgrade/`

升级所需的相关脚本统一放在该目录下。

## 脚本说明

### `check_code_status.sh`

- 运行位置：代码编译节点
- 运行方式：通常配置为周期性任务执行
- 输入参数：无
- 配置文件：`modules.conf`
- 作用：检测代码是否有更新；如有更新，则调用相关脚本打包并上传对应版本

### `transfer_packages.sh`

- 运行位置：代码编译节点或服务部署节点
- 运行方式：通常以调用方式执行
- 输入参数：`[upload/download] [filename]`
- 作用：上传版本到 `https://webdav.yeying.pub`；或下载版本到本地 `/opt/package` 目录

### `upgrade.sh`

- 运行位置：服务部署节点
- 运行方式：通常配置为周期性任务执行
- 输入参数：无
- 配置文件：`modules.conf`
- 作用：检测服务是否需要升级；升级时调用服务对应的升级脚本执行升级操作

### `upgrade_warehouse.sh`

- 运行位置：`warehouse` 服务部署节点
- 运行方式：通常以调用方式执行
- 输入参数：`[当前版本] [目标版本]`
